# Copy style (human)

When writing any user-facing copy (docs, UI text, README, changelog, headings, etc.):

- Don’t use em dashes (—). Use commas, periods, or parentheses.
- Minimal formatting: avoid heavy bolding; max one bold phrase per section if truly needed.
- Headings are sentence case (not Title Case).
- Avoid “AI voice” and filler (delve, leverage, robust, seamless, cutting-edge).
- Prefer plain language, short paragraphs, occasional contractions, less repetition.
- Keep meaning/facts the same; be concise.
